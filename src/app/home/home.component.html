Description

  • Créer un "signalement" avec les champs suivants:
    • Auteur:
    • Prénom : string - max 50 caractères
    • Nom : string - max 50 caractères
    • Date de naissance: date - pas plus de 100 ans
    • Sexe : Homme / Femme / Non-binaire
    • Email : email valide - unique
    • Description : string
    • Observations : array - chips - à choisir dans la liste provenant d’une API
  • Modifier un signalement
  • Liste des signalements

Pré-requis

  • Dernière version de Angular
  • Utiliser le moteur de rendu type material design
  • Mettre le code source sur Github.
  • Faire un README avec explication des choix technico-fonctionnels.

Compléments d’informations

  1. Le respect des normes de codage devra être appliqué.
  2. Une qualité de code sera attendue.
  3. Affichage de messages d’erreur (API, Formulaire, etc...)
  4. Une optimisation d’Angular en termes de performances est un plus.
Plus:
  • Proposer une belle UI.
  • Une ergonomie responsive.

Annexe

{{listData | json}}
